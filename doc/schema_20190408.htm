Setiap Request yang datang melalui API, harus di cocokan dengan token dalam table ini
    Flow Request Token :
  • User melakukan request access document ke CLIENT APPS
  • (CURL) CLIENT APPS merequest token pada EDOCS SYSTEM dengan menyertakan parameter username,fst_reff_source_code dan fst_reff_no (username yang dikirim harus terdaftar dalam EDOCS SYSTEM)
  • EDOCS SYSTEM akan mengenerate token dan memberikan token ke CLIENT APPS (JSON FORMAT)
  • setelah CLIENT APPS menerima token, maka CLIENT APPS akan meriderect page user ke alamat API EDOCS SYSTEM, dengan menyertakan parameter GET kode_token,username,fst_reff_source_code dan fst_reff_no
  • Ketika EDOCS SYSTEM meneriama request tersebut bila session_id kosong maka akan diisi dengan SESSION_ID saat ini, namum apabila session_id sudah terisi maka permission hanya akan diberikan kalau session_id yg di kirim dan session_id yg ada dlm table sama
